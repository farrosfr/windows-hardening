Strategic "Defense-in-Depth"

To achieve the highest level of content filtering and prevent bypasses, the NoA suite employs a multi-layered defense strategy. This guide explains how to maximize your protection.

Layer 1: OS Adapter DNS

The first line of defense is at the Operating System level. By setting DNS servers directly on every network adapter, we ensure every request from the computer is filtered.

Recommended Config:

$dnsIpv4 = @("208.67.222.123", "208.67.220.123")
$dnsIpv6 = @("2620:119:35::123", "2620:119:53::123")

Why OpenDNS (Umbrella)? Cisco's FamilyShield is industry-renowned for its aggressive categorization and zero-config blocking of malicious/adult content.

Layer 2: Browser DoH Lock

Modern browsers often ignore OS DNS by using DNS over HTTPS (DoH). We use Registry Policies to "Lock" the browser to our secure provider.

"We force the DoH mode to secure and point it to OpenDNS, preventing manual bypasses in settings."

Layer 3: SafeSearch Enforcement

DNS blocks domains, but SafeSearch filters results. We modify the hosts file to perform a local hijack of search engine domains.

216.239.38.120 www.google.com
216.239.38.120 google.com
204.79.197.220 www.bing.com

Summary: Why This Works

Combining these layers creates a Maximum Guard environment:

  • DNS blocks the "Roads" (Domains).
  • Policies lock the "Cars" (Browsers) to those roads.
  • Hosts File ensures "Windows" (Search Results) stay clean.

Refer to harden.ps1 in the root folder to apply these changes.
